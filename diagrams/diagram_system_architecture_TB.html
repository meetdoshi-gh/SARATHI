SARATHI — End-to-End System Architecture
EnvironmentMetaDrive Simulator
Single fixed map · no traffic · continuous action space
POMDP
MetaDrive provides a composable road network. The POMDP formulation strips velocity, heading & acceleration — leaving only visual perception and a nav hint to the checkpoint.
SensorsStereo Cameras + Nav Vector
Left & Right · 120×160 · Grayscale · 65° FOV · Navigation 10-dim
No velocity · No heading
Two hood-mounted virtual cameras. Grayscale reduces per-frame memory 3× vs RGB. The 10-d nav vector gives bearing & distance to checkpoint only — no dynamics.
PerceptionShared CNN Encoder ×2
Runs on left & right frames independently · 120×160→128-d · 3 blocks · GroupNorm · MaxPool
72 K params
Same 72K-param encoder runs on both frames (shared weights). GroupNorm is batch-size-independent — important for RL's small, shifting mini-batches. Returns two 128-d feature vectors.
FusionFeature Concatenation
[left 128 ‖ right 128 ‖ nav 10] → 266-d fused representation
InputEncoderLR
Simple concatenation — no learned fusion gate. Fully differentiable, adds zero parameters. Actor and each critic hold separate encoder instances to isolate gradient signals.
Policy · SAC / RSACActor + 2 Critics
Actor FC(512,128)→µ,σ · Critics FC(512,128)→Q · α auto-tune · τ=0.025 · RSAC adds LSTM(h=128)
From scratch
Clipped double-Q (min of Q1, Q2) suppresses overestimation. Entropy auto-tuning targets −action_dim. RSAC inserts LSTM(h=128) before output heads, processing T=16 step sequences.
OutputContinuous Actions
Steering: −15°→+15° · Engine force: 0.1→0.125 · tanh-squashed Gaussian
2-dim continuous
Sampled from squashed Gaussian during training (reparameterization trick). At eval, deterministic mean tanh(µ) gives stable performance — achieved 99.23% route completion.
FeedbackReward + Next Observation
+100 destination · +Δd progress · +0.5v speed · −10 off-road / sidewalk
→ Replay Buffer
Dense progress and speed rewards provide per-step signal. Sidewalk penalty prevents the failure mode where the agent drifts laterally before triggering the off-road terminal event.
RSAC
Extension
In Recurrent SAC (RSAC), an LSTM (h=128) is inserted between the fused feature and the actor/critic heads. The flat replay buffer is replaced by an episode-based buffer sampling contiguous fragments of T=16 steps, preserving the temporal coherence the LSTM requires. Built from scratch — not supported by standard RL libraries.
